# 🛒 Shopper Spectrum
### Customer Segmentation & Product Recommendation System

> An end-to-end machine learning project that segments e-commerce customers using RFM analysis + KMeans clustering and recommends similar products using Item-Based Collaborative Filtering — deployed as an interactive Streamlit application.

---

## 🎯 Problem Statement

Modern e-commerce businesses generate vast amounts of transaction data but struggle to act on it. This project solves two core problems:

1. **Customer Segmentation** — Group customers into actionable segments (High-Value, Regular, Occasional, At-Risk) using RFM feature engineering and KMeans Clustering, enabling targeted marketing strategies for each group.

2. **Product Recommendation** — For any given product, return the top 5 most similar products using Item-Based Collaborative Filtering (cosine similarity on a CustomerID × Product purchase matrix).

## 🧠 Machine Learning Pipeline

### Feature Engineering — RFM

| Feature | Definition | Formula |
|---|---|---|
| **Recency** | Days since last purchase | `snapshot_date − max(InvoiceDate)` |
| **Frequency** | Number of unique orders | `count(unique InvoiceNo)` |
| **Monetary** | Total spend | `sum(Quantity × UnitPrice)` |

### Preprocessing

- Removed cancellations (InvoiceNo starting with `C`)
- Dropped rows with missing CustomerID (~25% of raw data)
- Filtered invalid Quantity ≤ 0 and UnitPrice ≤ 0
- Applied **log transformation** on Frequency and Monetary (right-skewed)
- Applied **StandardScaler** (zero mean, unit variance)
- Applied **Winsorization** (3×IQR cap) on Monetary outliers

### Clustering Models

| Model | Silhouette ↑ | Davies-Bouldin ↓ | Notes |
|---|---|---|---|
| **KMeans (K=4)** ✅ | Best | Best | Final model — fast, interpretable, deployable |
| Agglomerative (Ward) | Competitive | Competitive | Good but slow at inference |
| DBSCAN | Variable | N/A | Produces noise points — not suitable for deployment |

**Optimal K selected via:** Elbow Method + Silhouette Score analysis.

### Customer Segments

| Segment | Recency | Frequency | Monetary | Strategy |
|---|---|---|---|---|
| 🌟 **High-Value** | Low | High | High | Loyalty rewards, VIP access |
| 📊 **Regular** | Medium | Medium | Medium | Upsell, cross-sell |
| ⚠️ **Occasional** | Medium-High | Low | Low | Re-engagement campaigns |
| 🔴 **At-Risk** | High | Very Low | Very Low | Win-back offers |

### Recommendation System

- Built a **CustomerID × Product Description** pivot table (top 500 products)
- Computed **Item-Item Cosine Similarity** matrix (500 × 500)
- For any product query → return top N most similar products by similarity score
- Fuzzy matching handles partial product name inputs
